# Final Replit Push -- Make This Thing Sing

This is a major experience upgrade based on a council of 5 behavioral design frameworks (Switch, Power of Moments, Hooked, Building a StoryBrand, Reality is Broken) reviewing the current product. Every change here is intentional. Don't simplify or skip things because they seem ambitious. I want you to swing for the fences on this one.

Read this entire prompt before starting. Then execute section by section.

## 1. Kill "No Nudges Yet" -- The First Mission Happens Immediately

This is the single most important change. Right now a user finishes the assessment, lands on the dashboard, and sees "No nudges yet -- Your first learning nudge arrives on Monday." That kills all momentum.

**What to build:**

When the assessment completes and scores are calculated, the system must immediately generate the user's first skill challenge (not on Monday, NOW). This means:

- The assessment scoring logic triggers an immediate call to generate the first challenge for the user's highest-priority skill (the "Your First Move" skill)
- This challenge appears directly on the results page, below the skill map, as a clear call to action: "Ready for your first move?" with a button to start
- The challenge is the same format as weekly challenges (one idea, one use case, one action, one reflection, one story) but positioned as "Your First Move" with special visual treatment
- After this first challenge, the weekly schedule kicks in normally
- Also: if a user finishes a challenge and wants to keep going, add a "Keep Going" button that immediately generates and shows the next challenge. The weekly schedule is a floor, not a ceiling. Motivated people should never be gated by a calendar.

**Database change:** Add a `is_first_challenge` boolean to the nudges table so the first one can be styled differently.

**Important naming note:** Throughout the app, stop calling these "nudges." That's internal behavioral design language, not user-facing language. For now, use "missions" or "skill challenges" as placeholder copy in the UI. The final naming will be decided separately, but the codebase should use a generic term that's easy to find-and-replace later. Variable names can stay as `nudge` internally, but all user-facing strings should say "mission" or "challenge."

## 2. The Results Reveal -- Full Production Cinematic Experience

The results page currently shows data. It should create an EMOTION. This is the single moment users will screenshot and share. Design it like an event.

**The reveal sequence (timed, animated):**

1. Assessment ends. Screen transitions to a "Calculating your results..." state with a subtle loading animation (not a spinner -- something branded, like the Electric Thinking colors pulsing or nodes appearing)
2. Brief pause (2-3 seconds of anticipation)
3. The skill map BUILDS itself, node by node, from Level 0 upward. Each mastered skill lights up green with a small celebration micro-animation as it appears. In-progress skills appear as yellow pulses. Locked skills appear dimmed or fogged. This should take about 10-15 seconds total. It should feel like watching your journey draw itself.
4. As the map builds, a count-up appears: "12 of 25 skills mastered" counting up from 0 to the final number
5. After the map is built, the level title appears with weight and ceremony. NOT "You scored Level 3." Instead: "You're a Level 3: Specialized Teammates" with the level name large, styled, and identity-forward
6. Below the level title, show a "Signature Skill" -- the single skill where the user demonstrated the most depth during the assessment. Frame it as: "Your Signature Skill: Instruction Design -- you showed real depth here, and it's rare." This makes the user feel individually seen, not just categorized
7. Below that, a "Bright Spots" section -- 2-3 sentences (generated by Claude during scoring) that tell the user what they're already doing well. Lead with strengths, not gaps. "You've built strong habits around first drafts and output editing. Most people at your level haven't figured out voice-first capture yet -- that's your unlock."
8. Below that, a "Future Self" paragraph -- what does the next level look and feel like for someone in their role? "As a Level 4 Agentic Workflow user, you won't just use AI tools -- you'll design systems that run without you. Imagine your weekly reporting happening automatically while you focus on strategy." This is the destination postcard.
9. THEN show "Your First Move" with the immediate challenge (from Section 1 above)
10. At the bottom, a shareable results card -- a beautifully designed image (auto-generated, branded, shows their level, signature skill, and skill map) with a "Share on LinkedIn" and "Copy Link" button

**The "Bright Spots Brief" and "Future Self" paragraph:** These are generated by Claude Opus during the scoring step. Add two new fields to the assessment scoring output: `bright_spots_text` (2-3 sentences about what the user does well) and `future_self_text` (2-3 sentences painting the picture of their next level). Store these in the assessments table. The scoring prompt needs to be updated to generate these alongside the skill scores.

**The Signature Skill:** During scoring, Claude identifies which skill the user demonstrated the most depth/sophistication on. Store as `signature_skill_id` in the assessments table.

**The shareable card:** Generate a branded image (or a styled HTML card that can be screenshotted) showing: Electric Thinking logo, the user's level and level name, their signature skill, their mastered skill count, and the skill map in miniature. Style it with the brand colors (warm beige background, pink accents, Tomorrow Bold font). Include Open Graph meta tags so when the link is shared on LinkedIn/Slack, it shows a rich preview.

## 3. RPG Skill Map -- Make It a Game World

The current skill map is functional but looks like a chart. It needs to feel like a world you want to explore.

**Visual overhaul:**

- **Regions, not rows.** Each level should be a distinct visual region on the map with its own color personality and atmosphere. Level 0 (Foundations) could be a starting village/basecamp. Level 1 (Accelerator) is an open road. Level 2 (Thought Partner) is a more complex landscape. Level 3 (Specialized Teammates) is a workshop/forge. Level 4 (Agentic Workflow) is a command center or tower. Use the brand color palette to differentiate: each region gets a primary accent color.
- **Nodes are destinations, not dots.** Each skill node should be a small illustrated icon or badge, not a circle. Mastered nodes glow green with a subtle pulse. In-progress nodes have a yellow aura and animate gently (beckoning). Locked nodes are dimmed, slightly fogged, with a subtle "?" or lock icon.
- **Paths connect nodes.** Visible paths/roads between nodes that light up as you progress. When a skill is mastered, the path to the next node illuminates.
- **Fog of war.** Levels beyond your current level + 1 are fogged. You can see the outline of the region but not the details. When you level up, the fog dramatically lifts on the next region with an animation.
- **"You are here" indicator.** A glowing marker or character icon on your current active skill that pulses. This is your position in the world.
- **Exploration.** Users can click on ANY visible node (including locked ones in the next level) to see a preview card: skill name, what it means, what it looks like in practice, and what you need to unlock it. Fogged nodes show just a mysterious silhouette with "Keep going to discover this skill."
- **Level-up animation.** When a user completes all 5 skills in a level, the fog on the next region lifts dramatically. New nodes appear with a cascade animation. The level name appears large. This should feel like unlocking a new area in a game. Add a celebration effect (confetti, particles, glow burst, whatever fits the brand).
- **Progress framing.** Always emphasize distance traveled, not remaining. Show "12 of 25 skills mastered" not "13 remaining." Show completed paths glowing behind you. The visual weight should be on what you've accomplished.

**Technical note:** This is a significant frontend component. Build it as a self-contained React component (`SkillMap` or `ProgressionMap`) with its own animation logic. Use CSS animations and/or a lightweight animation library (Framer Motion is ideal if available). The map data comes from the user's skill statuses via API. The component should be responsive (works on mobile too, perhaps as a vertical scroll on small screens).

I want this to be genuinely impressive. Push the visual design hard. Think Duolingo's skill tree meets a fantasy RPG map. If the first version looks like a chart with better colors, that's not enough. It should make someone say "whoa" when they see it.

## 4. Team Layer -- Social Proof That This Is a Team Thing

The product is built for enterprise teams but the experience is entirely solo. Fix that.

**After assessment, show a team snapshot:**
- "You're the [Nth] person on [Team Name] to complete the assessment"
- If they're the first: "You're the first on your team! Share the link to get your teammates started." with an invite link
- If others have completed: "Your team's average level is [X]. You're at Level [Y]." Show a simple visual of team distribution across levels (small bar chart or dot plot).
- Show this on the results page, below the shareable card

**On the dashboard, add a Team Activity feed:**
- "Sarah just hit Level 2!" (anonymized or named based on org privacy settings)
- "[X]% of your team has mastered [Skill Name]" shown on each skill node tooltip
- "Most people at your level complete this skill in about 2 weeks" as benchmark text on the active skill

**For users who are the only person on their team (beta testers, early users):**
- Show "Team: 1 of 1" with a playful tone: "Just you so far. This gets even better with your team." and an invite link
- Don't hide the team features, make them visible but in a "coming soon for you" state that creates desire

**Database/API:** The team snapshot data already exists (users belong to orgs, assessments have scores). This is a display layer change, not a data model change. Create an API endpoint that returns: team member count, assessment completion count, average level, level distribution, and recent completions.

## 5. Investment Moment -- The User Commits

After results (and after seeing the team snapshot), the user makes two choices:

**Choice 1: Commitment level.** Don't call it "nudge frequency." Frame it as a commitment to their growth journey. Use a slider or selection:
- "How fast do you want to level up?"
- Options presented as a fun, gamified scale: once a week, twice a week, every other day, daily
- Default to weekly. But make the higher options feel exciting and aspirational, not overwhelming
- "Most people start weekly and speed up as they get hooked" as social proof text
- Store this as the user's challenge delivery frequency

**Choice 2: Preferred day/time.** If they chose weekly, when? "When's the best time for a skill challenge?" Quick picker.

**This replaces the current static nudge_day field.** The frequency is new (currently hardcoded to weekly). Add a `challenge_frequency` field to the users table (values: weekly, twice_weekly, every_other_day, daily). Update the scheduled job to respect this frequency per user.

Frame the entire investment moment as: "Set up your journey" -- it's a 30-second step that creates ownership. The user chose this. They committed. That makes the first challenge feel like something they signed up for, not something that was assigned.

## 6. Admin Testing Mode

I need to be able to walk through every screen and moment in under 10 minutes, not over 10 days. Add a "Testing" section to the admin panel with these buttons:

1. **"Generate challenge now"** -- immediately generates and delivers the next skill challenge for a selected user (bypasses the schedule)
2. **"Simulate skill completion"** -- marks a specific skill as complete for a user, triggers all downstream effects (dashboard update, next skill assignment, streak update, level-up if applicable)
3. **"Trigger level-up"** -- simulates completing all skills in a level for a user, triggers the full level-up experience (animation, email, badge)
4. **"Preview challenge email"** -- renders what the next challenge email looks like for a user without sending it
5. **"Reset user"** -- clears a test user's assessment, skill data, challenges, and badges so I can run through the full journey from scratch
6. **"Send test email"** -- sends any system email (welcome, challenge, level-up, re-engagement) to a specified email address for preview

## 7. Scoring Prompt Update

The assessment scoring prompt (the one that analyzes the transcript and assigns Red/Yellow/Green per skill) needs to also generate:

1. `signature_skill_id` -- the skill where the user showed the most depth
2. `signature_skill_rationale` -- one sentence explaining why (shown to user as "you showed real depth here")
3. `bright_spots_text` -- 2-3 sentences about what the user is already doing well, framed as strengths and advantages, not checkboxes
4. `future_self_text` -- 2-3 sentences painting a vivid, personalized picture of what the next level looks and feels like for someone in their specific role
5. `trigger_moment` -- if mentioned during the assessment, when/where the user typically reaches for AI (used for personalized challenge delivery timing)

All of these get stored in the assessments table (add the new columns) and displayed on the results page.
