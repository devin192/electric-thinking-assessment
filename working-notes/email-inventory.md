## What Changed in This Rebuild

The templates were converted from CSS-class-based layout (using `<style>` blocks) to proper table-based HTML email layout with inline styles. Specific changes:

1. **Layout:** All content now uses `<table role="presentation">` for layout instead of `<div class="card">`. This ensures correct rendering in Outlook's Word-based rendering engine.

2. **Styles:** All critical styles are inline on each element. The `<style>` block is kept only for web font import and dark mode progressive enhancement (clients that support it will use it, those that don't ignore it).

3. **CTA Buttons:** Built as table-based buttons with VML fallback for Outlook. Each button includes `<!--[if mso]>` conditional comments for proper rendering in Outlook.

4. **Outlook Support:** Added `<!--[if mso]>` wrapper around the main container table for fixed-width rendering. Added `mso-line-height-rule: exactly` on headings.

5. **Dark Mode:** Added `color-scheme: light dark` meta tags and `@media (prefers-color-scheme: dark)` rules for clients that support it (Apple Mail, iOS Mail).

6. **Unsubscribe Headers:** All recurring emails now include both `List-Unsubscribe` and `List-Unsubscribe-Post` headers for one-click unsubscribe compliance.

7. **Brand Consistency:** Level-up and welcome emails use the correct level color from the brand palette (cyan for 0, gold for 1, pink for 2, orange for 3, blue for 4) instead of a generic class.

8. **Font Fallbacks:** Headers use `'Tomorrow', 'Trebuchet MS', Arial, sans-serif`. Body text uses `-apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, Arial, sans-serif`. These stacks degrade gracefully across all email clients.
