## 3. Data Model (shared/schema.ts)

15 tables total. Key relationships:

### Core entities

**organizations** -- Multi-tenant org container
- id, name, industry, size, settingsJson, createdAt

**users** -- All user types (member, manager, org_admin, system_admin)
- id, orgId (FK organizations), email (unique), name, password, roleTitle, aiPlatform
- userRole (member/manager/org_admin/system_admin), nudgesActive, nudgeDay
- challengeFrequency (weekly/twice_weekly/every_other_day/daily)
- timezone, onboardingComplete, emailValid
- emailPrefsNudges, emailPrefsProgress, emailPrefsReminders, unsubscribeToken
- createdAt

**levels** -- 5 AI fluency levels (Foundations through Agentic Workflow)
- id, name, displayName, sortOrder (0-4), description, visualTheme

**skills** -- 25 skills (5 per level)
- id, levelId (FK levels), name, description, sortOrder (0-24), isActive

### Assessment

**assessments** -- Assessment sessions and results
- id, userId (FK users), transcript, contextSummary
- assessmentLevel, activeLevel, scoresJson, firstMoveJson
- signatureSkillId (FK skills), signatureSkillRationale
- brightSpotsText, futureSelfText, triggerMoment
- status (in_progress/completed), startedAt, completedAt

**assessmentQuestions** -- Predefined assessment questions (not heavily used; AI drives conversation)
- id, skillId (FK skills), questionText, questionType, scoringLogicJson, sortOrder

### Progression

**userSkillStatus** -- Per-user per-skill status tracking
- id, userId (FK users), skillId (FK skills)
- status (red/yellow/green), explanation, completedAt, streakCount
- Unique constraint on (userId, skillId)

**verificationAttempts** -- Quiz attempts to verify yellow -> green
- id, userId, skillId, questionsJson, answersJson, passed, attemptedAt

**badges** -- Earned achievements
- id, userId (FK users), badgeType (skill_complete/level_up/streak/ultimate_master)
- badgeDataJson, earnedAt

### Nudges / Learning

**nudges** -- AI-generated learning challenges
- id, userId (FK users), skillId (FK skills)
- contentJson ({opener, idea, use_case, action, reflection, story, subject_line})
- subjectLine, emailId, emailSent, emailOpened, inAppRead
- isFirstChallenge (boolean), sentAt, createdAt

**nudgeVoiceGuide** -- System prompt for nudge generation
- id, promptText, updatedAt

### Organization

**invites** -- Org invite tokens
- id, orgId (FK organizations), email, invitedBy (FK users)
- token (unique), accepted, expiresAt, createdAt

**liveSessions** -- Scheduled live training sessions
- id, levelId (FK levels), title, description, sessionDate, joinLink, recordingLink

### System

**aiPlatforms** -- Supported AI platforms (ChatGPT, Copilot, Gemini, Claude, Other)
- id, name, displayName, isActive, sortOrder

**systemConfig** -- Key-value config store
- key (PK), value

**activityFeed** -- Org-level activity events
- id, orgId, userId, eventType, eventDataJson, createdAt

**emailLogs** -- Email delivery tracking
- id, userId, emailType, resendId, recipientEmail, event, metadata, createdAt

## 5. Server Modules

### assessment-ai.ts
- `getAssessmentResponse(messages, userId)`: Fetches system prompt from DB config key `assessment_conversation_guide`, injects user context (name, role, platform), calls Claude claude-sonnet-4-20250514 with max_tokens 800. Returns AI assistant message.
- `scoreAssessment(transcript, userId)`: Builds full framework description from DB levels/skills, sends transcript with detailed scoring instructions to Claude claude-sonnet-4-20250514 (max_tokens 4000). Returns: per-skill scores (red/yellow/green + explanation), assessmentLevel, activeLevel, contextSummary, firstMove (skillName + suggestion), signatureSkillName, signatureSkillRationale, brightSpotsText, futureSelfText, triggerMoment.

### nudge-ai.ts
- `generateNudge(userId, skillId)`: Uses nudge voice guide as system prompt, includes user context + assessment contextSummary + previous nudges for dedup. Calls Claude claude-sonnet-4-20250514 (max_tokens 1500). Returns: {opener, idea, use_case, action, reflection, story, subject_line}. Has fallback content if JSON parsing fails.
- `generateVerificationQuestions(skillId, userId)`: Generates 3 MCQ questions personalized to user context. Calls Claude claude-sonnet-4-20250514 (max_tokens 2000). Returns: array of {question, options[], correctIndex, explanation}. Has fallback questions if parsing fails.

### elevenlabs.ts
- `getConversationSignedUrl()`: Reads agent ID from systemConfig, calls ElevenLabs REST API with ELEVENLABS_API_KEY to get a signed WebSocket URL. The actual WebSocket handling is client-side.

### email.ts
- `baseTemplate(content)`: HTML email wrapper with Electric Thinking branding (Tomorrow + Source Sans 3 fonts, warm beige background #FFF8F0, pink accent #FF2F86).
- 9 email functions: sendNudgeEmail, sendWelcomeEmail, sendSkillCompleteEmail, sendLevelUpEmail, sendReEngagementEmail, sendReAssessmentEmail, sendAbandonedAssessmentEmail, sendInviteEmail, sendManagerOnboardingEmail (3-step sequence).
- All emails check user preferences (emailValid, emailPrefsNudges/Progress/Reminders).
- All include unsubscribe links and List-Unsubscribe headers.
- Uses raw HTML strings, NOT React Email components.

### auth.ts
- Session-based auth using express-session with connect-pg-simple (PostgreSQL session store).
- bcryptjs password hashing (12 salt rounds).
- `hashPassword(pw)`, `comparePasswords(pw, hash)`.
- `setupAuth(app)`: Configures session middleware. Cookie: maxAge 30 days, httpOnly, secure: false, sameSite: lax.
- `requireAuth(req, res, next)`: Middleware checking session.userId.
- `requireAdmin(req, res, next)`: Middleware checking userRole === "system_admin".
- NO OAuth (Google or Microsoft). Only email/password.

### storage.ts
- `DatabaseStorage` class implementing `IStorage` interface.
- Full CRUD for all 15 tables.
- Notable methods: `getActiveNudgeUsers()`, `getAbandonedAssessments()`, `upsertUserSkillStatus()`, `getUnsentNudges()`, `getConsecutiveUnopenedNudges()`, `resetUserProgress()`, `getAnalytics()`, `getTeamAnalytics()`, `getUserByUnsubscribeToken()`.

### cron.ts
4 scheduled jobs:
1. **Nudge generation** (2am daily): Generates nudges for active users with yellow skills. Respects per-user challengeFrequency. Has cost threshold config (`max_nudge_cost_daily`).
2. **Nudge delivery** (3pm daily): Sends unsent nudges via email. Logs to emailLogs.
3. **Daily checks** (10am daily): 3+ weeks of unopened emails triggers re-engagement email. 6+ weeks pauses nudges. 24h+ abandoned assessments get reminder email.
4. **Re-assessment reminders** (quarterly check): 90+ days since last assessment triggers reminder email.

### badge-svg.ts
- `generateBadgeSVG(badge)`: Creates 600x400 SVG badges for badge types: skill_complete, level_up, streak, ultimate_master. Uses brand fonts and gradient backgrounds.

### resend-client.ts
- `getUncachableResendClient()`: Gets Resend API key via Replit Connectors API (HTTP call to REPLIT_CONNECTORS_HOSTNAME), not a direct env var. Returns {client, fromEmail}.

### seed.ts
Seeds database with:
- 5 levels (Foundations, Accelerator, Thought Partner, Specialized Teammates, Agentic Workflow)
- 25 skills (5 per level)
- 5 AI platforms (ChatGPT, Copilot, Gemini, Claude, Other)
- Default assessment conversation guide (~35 lines of system prompt)
- Default nudge voice guide
- Admin user: admin@electricthinking.com / admin123
- Default ElevenLabs agent ID: agent_7501kjhd67qbeg19cb684bcj1ey2

## 9. System Prompts

### Assessment Conversation Guide (stored in systemConfig, key: "assessment_conversation_guide")
Default in seed.ts. Defines AI personality ("curious, warm, genuinely interested"), assessment approach (calibrate, adapt, use checklists for advanced users, keep it short for beginners), insight reframes ("trip over the truth"), deep context collection (job, tasks, workflow, concerns), conversation flow (concise, wrap up at 20min or 30 exchanges), and the full skill framework (5 levels, 25 skills by name).

### Assessment Scoring Prompt (inline in assessment-ai.ts)
Detailed scoring instructions for Claude. Given the transcript and framework, return JSON with: scores object (per skill: red/yellow/green + explanation), assessmentLevel (0-4), activeLevel, contextSummary, firstMove ({skillName, suggestion}), signatureSkillName, signatureSkillRationale, brightSpotsText, futureSelfText, triggerMoment.

### Nudge Voice Guide (stored in nudgeVoiceGuide table)
Default in seed.ts. Defines nudge generator personality. Used as system prompt when generating nudges.

### Nudge Generation Prompt (inline in nudge-ai.ts)
Given user context, skill details, assessment summary, and previous nudge titles, generate personalized learning challenge. Returns JSON: {opener, idea, use_case, action, reflection, story, subject_line}.

### Verification Question Prompt (inline in nudge-ai.ts)
Given skill details and user context, generate 3 MCQ questions with 4 options each. Returns JSON array of {question, options[], correctIndex, explanation}.

## 11. Auth Flow

1. **Register**: POST /api/auth/register with email, password, name. Password hashed with bcryptjs (12 rounds). User created, session set. Redirect to /onboarding.
2. **Login**: POST /api/auth/login with email, password. Password compared. Session set (userId stored in session). Cookie: 30-day maxAge, httpOnly, sameSite: lax, secure: false.
3. **Session check**: GET /api/auth/me reads session.userId, returns user or 401.
4. **Logout**: POST /api/auth/logout destroys session.
5. **Authorization**:
   - `requireAuth` middleware checks session.userId exists.
   - `requireAdmin` middleware checks userRole === "system_admin".
   - Manager routes check ["manager", "org_admin", "system_admin"] inline.
6. **No OAuth**: Google and Microsoft OAuth are NOT implemented. Only email/password auth exists.

## 12. Email System

### Infrastructure
- **Provider**: Resend, accessed through Replit Connectors API (not direct API key).
- **From address**: Dynamic from resend-client.ts response.
- **Template**: Single baseTemplate() function generating HTML email with brand styling.
- **Tracking**: Resend webhook at POST /api/webhooks/resend handles email.opened (updates nudge.emailOpened), email.bounced (marks user emailValid: false), email.complained (marks emailValid: false, pauses nudges).

### Email Types
1. **Welcome email** -- Sent on registration (not org users). Branded greeting.
2. **Nudge/challenge email** -- Weekly learning challenge. Shows opener, idea, action, reflection, story. Has branded "Read Challenge" CTA.
3. **Skill complete email** -- Congratulations on verifying a skill.
4. **Level-up email** -- Celebration of leveling up.
5. **Re-engagement email** -- Sent after 3+ weeks of unopened emails.
6. **Re-assessment email** -- Sent 90+ days after last assessment.
7. **Abandoned assessment email** -- Sent after 24h+ incomplete assessment.
8. **Invite email** -- Org invite with join link.
9. **Manager onboarding email** -- 3-step drip (welcome, set expectations, review analytics).

### Preferences
Users can control: emailPrefsNudges, emailPrefsProgress, emailPrefsReminders. All emails check the relevant pref before sending. Unsubscribe via token (no login required) or settings page.

## 13. Voice / Audio Integration

### ElevenLabs (Full-Duplex Voice Assessment)
- Server: `getConversationSignedUrl()` fetches a signed WebSocket URL from ElevenLabs API using the configured agent ID and API key.
- Client (assessment.tsx): Opens WebSocket connection to ElevenLabs. Captures user audio via MediaRecorder at 16kHz PCM16. Sends audio chunks over WebSocket. Receives agent audio + transcription events. Plays agent audio via AudioContext with Web Audio API.
- States: connecting, connected (listening/speaking), disconnected.
- Timeout: 20s connection timeout with retry/switch-to-text options.
- Transcript: Agent transcript displayed in real-time. On assessment complete, sends full transcript via __TRANSCRIPT_SAVE__ message to API.

### Audio Context (audio-context.ts)
Shared singleton AudioContext and MediaStream. Created during warmup page to ensure browser has already granted microphone permission and audio is initialized before the assessment page loads. Prevents the "click to allow microphone" interruption during the assessment.
